# 扉页：开源测评数字商品最小闭环

这份 Playbook 开源的不是一家收费网站，而是一套任何人都可以复刻的自营数字商品方法：从内容平台发现一个“有人想测”的主题，用 AI 制作题目和结果，生成测评链接，发布商品，买家付款后发送链接，交付完成。

## 一句话模型

```text
内容是销售员 → 平台商品是收银台 → 测评链接是交付物 → 结果页是消费体验
```

## 三个角色

| 角色 | 需要什么 | 提供什么 |
|---|---|---|
| 经营者 | 一个低成本、可重复制作的数字商品 | 选题、题目、网站、销售内容与链接交付 |
| 消费者 | 自我解释、身份表达、好奇与分享谈资 | 购买、答题和自愿分享 |
| 内容平台 | 持续产生能吸引用户的商品内容 | 需求信号、流量与现有交易能力 |

## 七步最小闭环

1. **看需求**：在内容平台记录高互动主题、用户原话和已有商品表达。
2. **定商品**：把人群、购买动机、结果期待和一句话承诺写进选题卡。
3. **做题目**：设计结果原型、场景题、计分和结果解释，输出标准 JSON。
4. **生成链接**：导入扉页，校验、试玩并得到完整测评链接。
5. **发布销售内容**：制作封面、标题、笔记正文和商品说明。
6. **发送链接**：买家完成平台订单后，人工或用平台允许的方式发送完整链接。
7. **记录结果**：只记录曝光、咨询、订单和退款等经营数据，不收集答题隐私。

## 第一版边界

- 不接支付，不做账号和订单系统；
- 不做一客一码，不验证买家身份；
- 不保存题目、答案或用户资料；
- 不限制链接转发；
- 不承诺盈利，不伪装心理、医学或命运诊断；
- 平台规则会变化，发布和交付前由经营者自行核对当期规则。

链接被转发是第一版接受的损耗。先验证能否完成“做出来—卖出去—发链接”的闭环，再决定是否增加服务器、兑换码或自动交付。
