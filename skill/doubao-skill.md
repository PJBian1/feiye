# 扉页 · 豆包出题、生成商品链接与交付

> 免费使用开源方法，用豆包、DeepSeek、Kimi、元宝或任何 AI 做一套测评，生成链接；经营者可以把自己的测评作为数字商品销售，买家付款后发送链接完成交付。

## 四步做出商品链接

1. 复制下面的完整提示词，最后把示例主题换成你的主题。
2. 把 AI 返回的标准 JSON 全部复制下来。
3. 打开「扉页」建站工作台，粘贴或上传 JSON，点击“校验并生成商品链接”。
4. 自己从封面完整答到结果，确认链接可用后保存为商品交付物。

创建页：<https://pjbian1.github.io/feiye/#/create>

## 出题提示词

```text
你是开源文学测评站「扉页」的出题人。请围绕指定主题创作一套可直接导入网站的文学原型测评。

【内容要求】
1. 设计 8—12 道题，每题 4 个选项；设计 4—8 个结果原型。
2. 先确定结果原型，再写题。题目必须落在具体场景与瞬间里，例如深夜、雨天、散场后、旧物、来信；不要直接问“你是否内向”。
3. 每个选项都没有高低好坏，并通过 scores 指向 1—2 个结果：主要倾向 2 分，次要倾向 1 分。每个结果至少获得 4 次主要得分机会，各结果覆盖尽量均衡。
4. 每个结果包含 name、source、quote、desc、traits、hue。desc 写 120—180 字：先肯定独特之处，再温柔点破隐秘软肋，最后用一句值得截图的话收尾。
5. quote 只有在能确认出处时才填写；不确定就写空字符串，禁止伪造名言。
6. result key 使用简短小写英文。所有字符串使用英文双引号，不要尾逗号。
7. 输出前自行检查：每个 scores key 都存在于 results；每个结果都能被测出；题目与结果数量符合要求。

【JSON 结构】
{
  "title": "测试标题，14 字以内",
  "subtitle": "一句有画面感的副标题",
  "tag": "分类 · 题数",
  "desc": "两三句邀请式开场白",
  "questions": [
    { "q": "题干", "options": [
      { "text": "选项文字", "scores": { "key1": 2, "key2": 1 } }
    ] }
  ],
  "results": {
    "key1": {
      "name": "原型名",
      "source": "出处",
      "quote": "真实引文或空字符串",
      "desc": "结果解读",
      "traits": ["词一", "词二", "词三"],
      "hue": 200
    }
  }
}

只输出一个标准 JSON 对象，不要 Markdown 代码围栏，不要解释，不要在 JSON 前后添加任何文字。

我的主题是：测测你是《红楼梦》里的谁
```

## 好主题的手感

- 你是某部作品里的谁：红楼梦、哈利·波特、百年孤独。
- 你是哪种人物谱系：宋朝文人、魏晋名士、侦探小说主角。
- 反向关系：哪位诗人会喜欢你、哪位作家会替你写日记。
- 非人物原型：你的灵魂乐器、深夜食堂人格、城市气质。

主题越具体，AI 越容易写出彼此有差异的结果。

## 最小销售与交付

1. 在内容平台发布围绕目标问题、使用场景和结果切片的内容；
2. 商品说明明确写出：买家得到一条在线测评链接、题目数量、预计时长和结果形式；
3. 买家使用平台现有交易能力完成订单；
4. 经营者发送完整链接；

## 常见问题

**JSON 报错怎么办？**

创建页会去掉代码围栏和尾逗号。仍然报错时，把错误原文发回 AI，要求它“只修复 JSON 格式，不改题目内容”。

**引文是真的吗？**

AI 可能编造文学名言。提示词已经要求“不确定就留空”，但发布前仍应人工核实 `source` 和 `quote`。

**题目存在哪里？**

题目经压缩后编码在商品链接的 `#` 后面，不上传数据库。链接在，题目就在。发送时必须保留 `#` 后的全部内容。

**给 Agent 使用的正式 Skill 在哪里？**

见 [`create-feiye-quiz/SKILL.md`](create-feiye-quiz/SKILL.md)。它包含字段规范和确定性校验脚本。
