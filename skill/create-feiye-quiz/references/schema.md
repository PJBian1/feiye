# 扉页测评 JSON 规范

## 目录

1. 顶层结构
2. 题目与评分

5. 限制与校验

## 1. 顶层结构

```json
{
  "title": "测试标题",
  "subtitle": "一句有画面感的副标题",
  "tag": "分类 · 十题",
  "desc": "两三句邀请式开场白",
  "questions": [],
  "results": {}
}
```

- `title`：必填，30 字以内；推荐 14 字以内。
- `subtitle`：可选，70 字以内。
- `tag`：可选，40 字以内。
- `desc`：可选，320 字以内。
- `questions`：必填，1—50 道；推荐 8—12 道。
- `results`：必填，2—24 个；推荐 4—8 个。

## 2. 题目与评分

```json
{
  "q": "雨停以后，你离开屋檐前会做什么？",
  "options": [
    {
      "text": "回头看一眼刚才躲雨的人",
      "scores": { "daiyu": 2, "stoner": 1 }
    }
  ]
}
```

- 每题使用 2—8 个选项，推荐固定 4 个。
- `q` 不超过 150 字，`text` 不超过 130 字。
- `scores` 的 key 必须存在于 `results`。
- 主要倾向用 2，次要倾向用 1；每个选项指向 1—2 个结果。
- 每个结果至少获得 4 次主要得分机会；各结果机会数尽量接近。
- 同一个问题的选项之间不得有明显的道德高低或正确答案。

## 3. 结果结构

```json
{
  "daiyu": {
    "name": "林黛玉",
    "source": "《红楼梦》 · 曹雪芹",
    "quote": "真实可核实的引文；不确定则留空",
    "desc": "120—180 字结果解读",
    "traits": ["敏锐", "自尊", "深情"],
    "hue": 338
  }
}
```

- result key 使用简短小写英文、数字或连字符。
- `name` 必填，24 字以内。
- `source` 60 字以内；`quote` 160 字以内；`desc` 650 字以内。
- `traits` 推荐 3 个，最多 6 个，每个 14 字以内。
- `hue` 是结果卡强调色，范围 0—360。
- 不确定文学原句的出处时把 `quote` 写成 `""`，禁止让 AI 补造名言。

## 4. 内容标准

- 先设计结果原型，确保原型之间有可描述的差异，再写题。
- 用深夜、雨天、旧物、散场、来信、饭桌、远行等可感知场景承载性格。
- 结果解读先肯定独特之处，再温柔指出软肋，最后落在可截图传播的一句。
- 避免精神疾病诊断、智力结论、性别刻板印象、阶层优劣和宿命断言。

## 5. 限制与校验

- 原始 JSON 最大 40KB。
- 所有字符串使用英文双引号。
- 不使用注释、Markdown 围栏或尾逗号。
- 运行 `scripts/validate_quiz.mjs`；只有退出码为 0 才可交付。
- 分享 URL 通过 deflate-raw 压缩并编码进 hash。链接超过 8,000 字符时应精简内容。
